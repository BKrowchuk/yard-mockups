TL;DR

  • Hollow core underscore uniqueness is a known gap; short-term fix is location-based selection, long-term requires an IPBS process change

  • Follow-up in 2–4 weeks
1

Current State — Pain Points

  • No swapping workflow — When a piece is taken from a different location there is no documentation. The next time someone looks for that piece it appears missing with no record of where it went.
  • Load sheets not syncing in real time — Paper/manual load sheets do not sync with Compass or IPBS.
  • Hollow core element identification — Underscore numbers repeat across levels, meaning there is no reliable way to distinguish which specific piece is being grabbed when multiples of the same underscore exist in the yard.
  • No audit trail — A panel may be picked but it's not necessarily the one tied to the load sheet, and there is no tracking in place.

Desired State — Key Decisions & Discussion

  • What counts as a swap — A swap only occurs when a piece is taken from a physically different row/location than assigned. Same-row picks are standard loading and do not trigger a swap.

Swapping flow in the app:

  1. Scan or open a load sheet → see all elements with their assigned row/location
  2. Click on an element → app displays a list of all locations where an instance of that element exists in the yard
  3. Operator selects the instance they are grabbing — if it's from a different location than assigned, the system automatically records it as a swap
  4. Element is marked as loaded and load sheet updates accordingly
  • Element selection & filtering — Location list must be sorted and easy to act on — no long scrolling. This also handles the hollow core underscore problem short-term since operators select by location, not underscore number.
  • Notifications & load sheet updates — Still to be defined: who gets notified on a swap, and whether the load sheet auto-updates in real time.

Hollow Core — Short Term

  • Show elements with locations in the selection list
  • Operators click to confirm which instance they grabbed
  • Works now, sets up traceability for the future

Hollow Core — Long Term

  • Make underscore numbers unique across the whole project in IPBS (not resetting per level)
  • Requires IPBS process change — not a blocker for current build
  • New elements only; will not affect pieces already in production
  • Edge cases to design for:
    • Piece assigned to a load sheet is no longer in the yard → flag and present next available nearby option
    • Swap initiated on an element already assigned to another active load
    • Swap initiated on an element not currently on any load
  • Racks — Shipping racks and other racks must be defined as distinct rack types. A "Clear Rack" function was discussed with multi-step verification to prevent accidental data loss. The system should also prevent a trailer from being released while it still has active unconfirmed piece assignments.

Wi-Fi & Yard Connectivity — Testing Plan

  • Connectivity has been a persistent blocker for yard app adoption; concrete pile layout and seasonal conditions disrupt access points frequently.
  • Test 1 — New directional access points between Plant 3 and the office. Installation pending second crane going live. More consistent coverage but shorter range — not a long-yard solution.
  • Test 2 — Starlink unit: ordered, expected setup ~2 weeks. To be trialled in a forklift or mounted in the yard. Anecdotal reports from staff using Starlink at home are encouraging.
  • Test 3 — Forklift-mounted 5G antenna: antenna positioned above the concrete piles to overcome signal blocking. Evaluated alongside Starlink.
  • Longer-term ideal: negotiate private tower access (Rogers Tower) for a dedicated network — by far the most reliable solution but complex to arrange.
  • Offline mode is being built into the app with a forced sync trigger after ~15 minutes before further actions are permitted.
  • Connectivity must be resolved before full yard app rollout — poor Wi-Fi was the primary reason initial yard app adoption failed.

Decisions Made

✓

Swapping in the Yard App will only be triggered when a piece is selected from a physically different row than originally assigned — swapping within the same row is not required.

✓

The load sheet UI will display all elements with their row/location and allow the loader to select from a location-sorted dropdown rather than requiring a manual swap screen.

✓

Full per-piece NCR traceability is deferred — it requires IPBS-level underscore numbering changes not feasible in the current iteration.

✓

Three connectivity tests (directional APs, Starlink, 5G antenna) will run concurrently over 2–4 weeks, with results reviewed before committing to an infrastructure path.

✓

Offline mode with a forced re-sync timer will be built into the Yard App as a resilience measure regardless of which connectivity solution is selected.

✓

Shipping racks and other racks will be treated as distinct rack types in the system.
